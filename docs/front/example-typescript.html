De HTML estático a JavaScript dinámico

Guía paso a paso que explica cómo transformar una página con datos escritos a mano en una página que carga sus datos desde un archivo JSON usando clases de JavaScript (TypeScript compilado).

1

El problema: datos mezclados con presentación

En la versión estática, el nombre del estudiante, su cédula y sus cursos están escritos directamente en el HTML. Para mostrar otro estudiante habría que editar el archivo HTML manualmente.

Versión estática — datos fijos en el HTML
<h1>Ana María Suarez</h1>

<tr>
  <td>Código</td>
  <td>10234320</td>    <!-- dato fijo -->
</tr>
Problema: Si hay 30 estudiantes necesitamos 30 archivos HTML diferentes, o editar el mismo archivo cada vez. Los datos y la presentación están acoplados.

La solución es separar los datos de la presentación: los datos van en un archivo JSON y el HTML queda como una plantilla vacía que JavaScript rellena.

2

El archivo de datos — student.json

JSON (JavaScript Object Notation) es un formato de texto para representar datos estructurados. Los navegadores y Node.js pueden leerlo directamente. 

{
  "name":     "Ana María Suarez",
  "code":     "10234320",
  "idNumber": "12221122",
  "age":      25,
  "address":  "Calle falsa 123",
  "phone":    "018002500",
  "courses": [
    { "name": "Ingeniería de SW",     "professor": "Pablo Picasso"    },
    { "name": "Futbol I",             "professor": "Juan Piña"        },
    { "name": "Algoritmos Genéticos",  "professor": "María de la Rosa"  }
  ]
}
Nota: los nombres de los campos (name, idNumber, etc.) están en inglés. Esta es una convención habitual en proyectos de software: los datos viajan en inglés y la interfaz los muestra en el idioma del usuario.

El campo courses es un arreglo de objetos. Cada objeto tiene dos campos: name y professor. Esto nos permite mostrar cualquier cantidad de cursos sin cambiar el HTML.

3

Las clases — Course y Person

Una clase es una plantilla que define la estructura de un objeto. El constructor es la función que se ejecuta cuando creamos una nueva instancia con new.

Clase Course

Representa un curso con su nombre y su profesor.

// TypeScript (src/person.ts)
class Course {
  name:      string;
  professor: string;

  constructor(name: string, professor: string) {
    this.name      = name;
    this.professor = professor;
  }
}
Clase Person

Representa al estudiante. El atributo courses es un arreglo de objetos Course — no de texto plano sino de instancias de la clase. 

// TypeScript (src/person.ts)
class Person {
  name:     string;
  code:     string;
  idNumber: string;
  age:      number;
  address:  string;
  phone:    string;
  courses:  Course[];   // arreglo de objetos Course

  constructor(name, code, idNumber, age, address, phone, courses) {
    this.name     = name;
    this.code     = code;
    this.idNumber = idNumber;
    this.age      = age;
    this.address  = address;
    this.phone    = phone;
    this.courses  = courses;
  }
}
¿Por qué clases y no objetos simples?
Las clases permiten verificar el tipo con instanceof, agregar métodos más adelante, y dejan claro en el código qué forma tiene cada dato. En TypeScript además el compilador detecta errores si intentamos usar un atributo que no existe.
4

Cargar los datos — loadPerson()

Esta función hace tres cosas: pide el archivo JSON al servidor, convierte el texto en objetos JavaScript, y construye instancias de nuestras clases. 

async function loadPerson(url) {

  // 1. Pide el archivo al servidor y espera la respuesta
  const response = await fetch(url);

  // 2. Convierte el cuerpo de la respuesta de texto JSON a objeto JS
  const data = await response.json();

  // 3. Por cada elemento del arreglo JSON crea un objeto Course
  const courses = data.courses.map(c => new Course(c.name, c.professor));

  // 4. Crea y devuelve el objeto Person con todos los datos
  return new Person(
    data.name, data.code, data.idNumber,
    data.age,  data.address, data.phone,
    courses
  );
}

async / await

fetch no devuelve los datos inmediatamente — hace una petición de red que tarda un tiempo indeterminado. async marca la función como asincrónica y await pausa la ejecución hasta que la promesa se resuelve, sin bloquear el navegador.

.map()

Transforma cada elemento de un arreglo aplicando una función. Aquí convierte cada objeto plano del JSON { name, professor } en una instancia real de Course. Devuelve un nuevo arreglo sin modificar el original.

5

El HTML como plantilla vacía

Los elementos que muestran datos ahora tienen un id y comienzan vacíos. JavaScript los encontrará por ese id y les inyectará el contenido.

Antes — dato fijo
<h1 class="fw-bold mb-4">
  Ana María Suarez
</h1>

<tbody>
  <tr><td>Código</td>
      <td>10234320</td></tr>
  <!-- … -->
</tbody>
Después — plantilla con id
<h1 id="student-name"
    class="fw-bold mb-4">
</h1>

<tbody id="datos-tbody">
  <!-- JavaScript inserta aquí -->
</tbody>
El HTML ya no tiene datos: solo tiene estructura (dónde van las cosas) e identificadores (cómo las encuentra JavaScript).
6

Renderizar en el DOM — renderPerson()

El DOM (Document Object Model) es la representación en memoria del HTML de la página. document.getElementById() busca un elemento por su id y devuelve una referencia a él para poder modificarlo. 

function renderPerson(person) {

  // textContent reemplaza el texto del elemento (sin interpretar HTML)
  document.getElementById('student-name').textContent = person.name;

  // innerHTML reemplaza el contenido HTML completo del elemento
  const datosTbody = document.getElementById('datos-tbody');
  datosTbody.innerHTML =
    row('Código',    person.code)     +
    row('Cédula',    person.idNumber) +
    row('Edad',      person.age + ' Años') +
    row('Dirección', person.address)  +
    row('Teléfono',  person.phone);

  // .map() convierte cada Course en un string de HTML, .join() los une
  const coursesTbody = document.getElementById('courses-tbody');
  coursesTbody.innerHTML = person.courses
    .map(c => courseRow(c.name, c.professor))
    .join('');
}

// Helpers: construyen el HTML de cada fila como texto
function row(label, value) {
  return '<tr class="border-top">' +
           '<td class="text-secondary">' + label + '</td>' +
           '<td>' + value + '</td>' +
         '</tr>';
}

textContent vs innerHTML

Usa textContent cuando el valor es texto puro (nombre, número). Usa innerHTML cuando necesitas insertar etiquetas HTML. textContent es más seguro porque no interpreta etiquetas.

.map().join('')

.map() transforma cada curso en un string HTML. El resultado es un arreglo de strings. .join('') los concatena en un solo string (sin separador) para asignarlo a innerHTML.

7

El arranque — type="module"

La etiqueta <script> tiene el atributo type="module". Esto cambia dos comportamientos importantes respecto a un script normal: 

<script type="module">

  // ... clases, loadPerson, renderPerson ...

  // Arranca todo: carga los datos y luego renderiza
  loadPerson('data/student.json')
    .then(person => {
      console.log('Cargado:', person);  // verificación
      renderPerson(person);
    })
    .catch(err => console.error(err));

</script>
Característicatype="module"Script normal
Espera al DOM Sí — corre después de que el HTML está listo No — puede correr antes de que existan los elementos
Scope Las variables son locales al módulo Las variables se agregan al objeto global window

.then() recibe la función a ejecutar cuando loadPerson termina. Si ocurre un error (archivo no encontrado, JSON inválido), .catch() lo captura.

8

El flujo completo

Cuando el navegador carga la página, ocurre esto en orden:

1
Navegador carga ejemplo-perfil-dinamic.html
Construye el DOM con el HTML estático. Los <tbody> están vacíos. El <h1> no tiene texto.
2
Ejecuta el <script type="module">
Define las clases Course y Person, define loadPerson y renderPerson, luego llama a loadPerson('data/student.json').
3
fetch pide data/student.json al servidor
El navegador hace una petición HTTP. El servidor responde con el texto del archivo JSON.
4
response.json() convierte el texto en objeto JS
.map() crea los objetos Course. new Person(...) construye el objeto final.
5
renderPerson(person) actualiza el DOM
getElementById localiza cada contenedor y textContent / innerHTML insertan el contenido. La página muestra los datos del estudiante.
